# 3초 생존 게임 기획서

## 1. 게임 개요
### 컨셉
- **장르**: 캐주얼 액션, 미니게임 컬렉션
- **플랫폼**: 모바일 웹 (반응형)
- **플레이 타임**: 1회 3초, 세션당 1~5분
- **핵심 메커니즘**: 무작위 미션을 3초 안에 성공하면 생존, 실패하면 게임오버
- **테마**: CRT 그린톤 레트로 아케이드

### 게임 플로우
```
홈 화면 → START → 미션 제시 (0.5초) → 플레이어 액션 (2.5초) → 성공/실패 판정 → 다음 미션 or 게임오버 → 결과 화면
```

## 2. 핵심 시스템

### 2.1 미션 시스템
각 미션은 다음 요소로 구성:
- **미션 텍스트**: 간결한 지시문 (한/영 지원)
- **미션 타입**: 135종의 고유 미니게임
- **난이도**: EASY, NORMAL, HARD
- **시각적 요소**: 색상, 아이콘, 애니메이션

매 라운드 미션 1개가 제시되며, 3초 내에 성공해야 다음 라운드로 진행.

### 2.2 입력 타입
- **TAP**: 단일 터치
- **SWIPE**: 방향 스와이프 (UP, DOWN, LEFT, RIGHT)
- **LONG_PRESS**: 1초 이상 롱프레스
- **MULTI_TAP**: 빠른 연타
- **DUAL_TAP**: 두 손가락 동시 터치

### 2.3 미션 카테고리 (135종)
- **시스템/해킹**: REBOOT, DEFRAG, KERNEL_PANIC, FIREWALL, BIOS_ERROR, SEGFAULT, BLUE_SCREEN 등
- **의료/생존**: HEARTBEAT, TRANSFUSE, ANTIDOTE, TOURNIQUET, QUARANTINE, DEFIBRILLATE, TRIAGE 등
- **전투/전술**: GRENADE_PIN, BARRICADE, SCOPE, RELOAD, CLAYMORE_AIM, TRIPWIRE, BREACH_CHARGE 등
- **통신/신호**: MORSE, BEACON, BROADCAST, SEMAPHORE, SIGNAL_INTERCEPT, FREQUENCY_JAM 등
- **전자/회로**: SOLDER, CAPACITOR, VOLTAGE_MATCH, CIRCUIT_BRIDGE, FIRMWARE, TUBE_REPLACE 등
- **환경/탐색**: COMPASS, GEIGER, DUST_STORM, WATER_LEVEL, FORAGE, TUNNEL_DIG, THERMAL_SCAN 등

### 2.4 난이도 시스템
- **스코어 기반 난이도 증가**
  - 0~5점: EASY 미션 (기본 16종)
  - 6~15점: EASY + NORMAL 미션 (84종)
  - 16~30점: NORMAL 미션 확장
  - 31점+: 전체 135종 (HARD 포함)

### 2.5 피드백 시스템
- **성공 시**:
  - 효과음 (2톤 사인파 880Hz + 1320Hz)
  - 화면 흔들림 (300ms) + 파티클 이펙트 (12개)
  - +1 점수 표시

- **실패 시**:
  - 효과음 (2톤 스퀘어파 200Hz + 150Hz)
  - 화면 붉은 플래시
  - "FAIL" 텍스트 + 최종 점수

## 3. UI/UX

### 3.1 화면 구성
- **홈 화면**: 최고 점수 표시, START 버튼, 설정 메뉴
- **게임 화면**: 타이머 바, 점수, 미션 텍스트, 미션 컴포넌트, 페이즈 뱃지
- **결과 화면**: 최종 점수, 신기록 표시, 재시작/홈 버튼

### 3.2 설정
- BGM 볼륨 조절
- SFX 볼륨 조절
- 언어 전환 (한국어/영어)

### 3.3 점수 저장
- 최고 점수 (localStorage)
- 최근 50개 점수 히스토리

## 4. 기술 스택
- **프레임워크**: Vue 3 (Composition API)
- **빌드 도구**: Vite
- **언어**: TypeScript
- **라우팅**: Vue Router 4
- **스타일**: CSS3 (커스텀 변수, Flexbox, Grid, Animation)
- **오디오**: Web Audio API (SFX 합성 + BGM 스텝 시퀀서)
- **폰트**: Mulmaru (커스텀 WOFF2)
- **상태 관리**: Vue Reactivity (Ref, Computed)
- **i18n**: 커스텀 동적 패턴 기반 번역 시스템

## 5. 메타 시스템

### 5.1 ENERGY (에너지)
플레이 횟수를 제한하여 자연스러운 재접속 루프를 유도한다.

- **최대치**: 30
- **충전 속도**: 5분당 1 (0→30 풀충전 = 2.5시간)
- **소모**: 게임 시작 시 5 소모 (풀 에너지 시 6판 가능)
- **캡 초과**: 일일 미션/이벤트 보상으로 30 초과 보유 가능
- **초과 시 규칙**: 자동충전 정지, 30 이하로 떨어지면 자동충전 재개
- **즉시 충전**: VOLT 소모 또는 리워드 광고 시청

### 5.2 VOLT (재화)
터미널 세계관에 맞는 게임 내 재화. 상점 구매 및 부활에 사용한다.

#### 획득 방법
- **드롭 방식**: 미션 클리어 시 확률적으로 VOLT 드롭
  - 기본 드롭 확률: 20% (1~3 VOLT)
  - 연속 10+ 클리어: 확률 30%로 상승
  - 연속 20+ 클리어: 드롭량 2~5 VOLT
  - 게임 흐름을 끊지 않는 팝업 연출
- **일일 챌린지**: 클리어 보상
- **리워드 광고**: 시청 시 VOLT 지급

#### 소비처
- 터미널 테마 구매 (상점)
- 에너지 즉시 충전
- 부활

> **밸런스 수치 (드롭 확률, 획득량, 가격)는 추후 플레이테스트를 통해 조정**

### 5.3 부활
게임오버 시 1회에 한해 이어하기 가능.

- **게임당 1회 제한**
- **부활 수단**: VOLT 소모 또는 리워드 광고 시청
- **부활 비용**: 추후 결정

### 5.4 상점 (OUTPOST TERMINAL SHOP)
VOLT를 사용하여 아이템을 구매하는 상점.

### 5.5 일일 챌린지
매일 동일한 미션 세트로 경쟁하는 별도 게임 모드.

- **홈 화면에서 별도 모드로 진입**
- **날짜 시드 기반**: 같은 날 같은 미션 순서 → 공정한 경쟁
- **서버 불필요**: 날짜를 시드로 사용하여 클라이언트에서 생성
- **보상**: VOLT

## 8. 구현 로드맵

```
Phase 1: 코어 메타 시스템
  - ENERGY 시스템 (충전/소모/캡초과)
  - VOLT 재화 (드롭/저장)
  - 부활 시스템

Phase 2: 상점
  - CRT 터미널 테마 구매
  - 에너지 충전
  - 홈 화면 개편

Phase 3: 일일 챌린지
  - 날짜 시드 기반 고정 미션 세트
  - 일일 최고기록 별도 저장

Phase 4: 랭킹
  - 서버 연동
  - 글로벌/일일 랭킹

Phase 5: 광고 SDK 연동
  - 리워드 광고 (부활, 에너지, VOLT)
  - 인터스티셜 광고
```
