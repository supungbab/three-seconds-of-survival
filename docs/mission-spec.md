# 미션 기획서 - 30개 미션 로드맵

## 현황

구현 완료: 9개 / 목표: 30개

---

## 난이도 시스템

### 스코어 구간별 해금

| 스코어 | 해금 난이도 | 라운드당 미션 수 |
|--------|-------------|------------------|
| 0~5    | EASY        | 1개              |
| 6~15   | NORMAL      | 2개              |
| 16~30  | NORMAL      | 3개              |
| 31+    | HARD        | 4개              |

### 다중 미션 제약

| 조건 | 제외 미션 | 사유 |
|------|-----------|------|
| 2개+ | DO_NOTHING, LONG_PRESS, HOLD_AND_TAP, RHYTHM | 시간 소진/타이밍 충돌 |
| 3개+ | SEQUENCE, SIMON, PATTERN_TAP, PINCH, ROTATE | 복잡도 과다 |
| 축소 | MULTI_TAP → tapCount 2~3, COUNT_TAP → 최대 3 | 시간 예산 |

## EASY (10개)

직관적 반응. 텍스트 읽고 바로 행동.

| # | ID | 이름 | 텍스트 예시 | 입력 | 상태 |
|---|----|------|-------------|------|------|
| 1 | `COLOR_TAP` | 색 탭 | "빨간색 탭!" | 탭 | ✅ 구현 |
| 2 | `SWIPE` | 스와이프 | "위로!" | 스와이프 | ✅ 구현 |
| 3 | `MULTI_TAP` | 2번 탭 | "2번 탭!" | 탭 | ✅ 구현 |
| 4 | `SIZE_TAP` | 크기 탭 | "큰 거 탭!" / "작은 거 탭!" | 탭 | ⬜ |
| 5 | `CATCH` | 잡아! | "잡아!" | 탭 | ⬜ |
| 6 | `SWIPE_MATCH` | 화살표 스와이프 | (화살표 아이콘만 표시) | 스와이프 | ⬜ |
| 7 | `DOUBLE_SWIPE` | 2번 스와이프 | "위로 2번!" | 스와이프 | ⬜ |
| 8 | `TAP_ZONE` | 영역 탭 | "왼쪽!" / "오른쪽!" | 탭 | ⬜ |
| 9 | `QUICK_TAP` | 빨리 탭 | "지금!" | 탭 | ⬜ |
| 10 | `DRAG_TO` | 드래그 | "끌어!" | 드래그 | ⬜ |

### 상세

#### 4. SIZE_TAP
- 화면에 크기가 다른 원 2~3개 표시
- "큰 거 탭!" 또는 "작은 거 탭!" 지시
- 틀린 원 탭 시 실패

#### 5. CATCH
- 원이 화면 내에서 천천히 이동
- 탭하면 성공
- 시간 내 못 잡으면 타임아웃 실패

#### 6. SWIPE_MATCH
- 텍스트 없이 큰 화살표 아이콘만 표시
- 화살표 방향대로 스와이프하면 성공
- SWIPE와 차이: 텍스트 대신 비주얼 판단

#### 7. DOUBLE_SWIPE
- 같은 방향으로 2번 스와이프
- 첫 스와이프 후 카운터 표시
- 다른 방향 스와이프 시 실패

#### 8. TAP_ZONE
- 화면을 좌/우 반으로 나눔
- "왼쪽!" 또는 "오른쪽!" 지시
- 반대쪽 탭 시 실패

#### 9. QUICK_TAP
- SHOWING 직후 타겟이 잠깐(1초) 나타났다 사라짐
- 나타나는 동안 탭하면 성공
- 사라진 후 탭하면 실패 (타겟 없음)

#### 10. DRAG_TO
- 시작점(원)과 도착점(빈 원) 표시
- 시작점에서 도착점까지 드래그하면 성공
- 드래그 궤적 시각화

## NORMAL (10개)

인지 부하 추가. 변환/매핑/카운팅 필요.

| # | ID | 이름 | 텍스트 예시 | 입력 | 상태 |
|---|----|------|-------------|------|------|
| 11 | `MULTI_TAP` | N번 탭 | "5번 탭!" | 탭 | ✅ 구현 |
| 12 | `REVERSE_SWIPE` | 반대 스와이프 | "반대로! 위로!" | 스와이프 | ✅ 구현 |
| 13 | `LONG_PRESS` | 길게 누르기 | "길게 눌러!" | 꾹 | ✅ 구현 |
| 14 | `ODD_ONE_OUT` | 다른 거 찾기 | "다른 거 탭!" | 탭 | ⬜ |
| 15 | `PATTERN_TAP` | 순서 탭 | "순서대로!" | 탭 | ⬜ |
| 16 | `PINCH` | 핀치 | "오므려!" / "벌려!" | 핀치 | ⬜ |
| 17 | `COUNT_TAP` | 세기 탭 | "몇 개? 탭으로!" | 탭 | ⬜ |
| 18 | `ROTATE` | 회전 | "돌려!" | 회전제스처 | ⬜ |
| 19 | `COLOR_SWIPE` | 색→방향 | "빨강=위!" | 스와이프 | ⬜ |
| 20 | `HOLD_AND_TAP` | 누르고 탭 | "꾹 + 탭!" | 복합터치 | ⬜ |

### 상세

#### 14. ODD_ONE_OUT
- 같은 모양/색 아이템 4~6개 중 하나만 다름
- 다른 하나를 탭하면 성공
- 변형: 색이 다름 / 모양이 다름 / 크기가 다름

#### 15. PATTERN_TAP
- 2~3개의 번호 붙은 점이 화면에 표시
- 1→2→3 순서대로 탭하면 성공
- 순서 틀리면 즉시 실패

#### 16. PINCH
- 두 손가락 오므리기(pinch-in) 또는 벌리기(pinch-out)
- 시각적 가이드: 두 원이 가까워지거나 멀어지는 애니메이션
- 반대 동작 시 실패

#### 17. COUNT_TAP
- 화면에 물건(별, 점 등) 2~5개 표시
- 물건 개수만큼 탭하면 성공
- 초과 탭 시 실패

#### 18. ROTATE
- 두 손가락으로 화면을 시계/반시계 방향으로 회전
- "시계방향!" 또는 "반시계!" 지시
- 일정 각도(90°) 이상 회전 시 성공

#### 19. COLOR_SWIPE
- 색상 하나 표시 + 색→방향 매핑 규칙 표시
- "빨강=위, 파랑=아래" 같은 규칙
- 표시된 색에 맞는 방향으로 스와이프

#### 20. HOLD_AND_TAP
- 한 손으로 왼쪽 영역 꾹 누른 상태에서
- 다른 손으로 오른쪽 영역 탭
- 꾹 누르기를 먼저 놓으면 실패

## HARD (10개)

인지 트릭, 기억, 복합 판단.

| # | ID | 이름 | 텍스트 예시 | 입력 | 상태 |
|---|----|------|-------------|------|------|
| 21 | `DUAL_TAP` | 양손 동시 탭 | "둘 다 탭!" | 양손탭 | ✅ 구현 |
| 22 | `DO_NOTHING` | 참기 | "누르지 마!" | 무입력 | ✅ 구현 |
| 23 | `COLOR_TAP_NEGATIVE` | 빼고 탭 | "빨간색 빼고 탭!" | 탭 | ✅ 구현 |
| 24 | `SEQUENCE` | 연속 동작 | "탭 → 위 → 아래" | 복합 | ✅ 구현 |
| 25 | `SIMON` | 기억 탭 | "기억해!" | 탭 | ⬜ |
| 26 | `FAKE_OUT` | 함정 | 가짜 지시 표시 | 판단 | ⬜ |
| 27 | `MATH_TAP` | 산수 | "3+4=?" | 탭 | ⬜ |
| 28 | `MIRROR_SWIPE` | 거울 스와이프 | "거울! 왼쪽!" | 스와이프 | ⬜ |
| 29 | `DUAL_SWIPE` | 양손 스와이프 | "↑↓ 동시!" | 양손스와이프 | ⬜ |
| 30 | `RHYTHM` | 리듬 탭 | "박자 맞춰!" | 타이밍탭 | ⬜ |

### 상세

#### 25. SIMON
- SHOWING 동안 2~3개 버튼이 순서대로 깜빡임
- ACTING에서 같은 순서로 탭하면 성공
- 순서 틀리면 즉시 실패

#### 26. FAKE_OUT
- 화면에 2개 지시가 동시 표시
- 하나는 진짜(초록 테두리), 하나는 가짜(빨간 취소선)
- 가짜 지시를 수행하면 실패
- 변형: 가짜가 더 크게/눈에 띄게 표시되어 함정

#### 27. MATH_TAP
- 간단한 산수 표시 (ex: "3+4", "9-5")
- 답 선택지 3개 중 정답 탭
- 오답 탭 시 실패

#### 28. MIRROR_SWIPE
- "거울!" 프리픽스 + 방향 지시
- 좌우만 반전 (위/아래는 그대로)
- "거울! 왼쪽!" → 오른쪽으로 스와이프
- REVERSE_SWIPE(상하좌우 전부 반전)와 차별화

#### 29. DUAL_SWIPE
- 양손으로 반대 방향 동시 스와이프
- "↑↓" → 왼손 위, 오른손 아래
- 같은 방향 스와이프 시 실패
- 멀티터치 2개의 스와이프 방향 각각 검증

#### 30. RHYTHM
- 리듬 가이드(원이 수축하는 타이밍 표시) 2~3비트
- 각 비트 타이밍에 맞춰 탭 (±200ms 허용)
- 타이밍 벗어나면 실패
- 다중 미션에서 제외 (시간 소진형)

## 구현 우선순위 제안

### Phase 1 - 변형 추가 (기존 입력 재활용)
빠르게 구현 가능. 새 입력 감지 불필요.

1. `TAP_ZONE` — 탭 x/y 좌표로 좌우 판별
2. `SIZE_TAP` — COLOR_TAP 구조 재활용
3. `DOUBLE_SWIPE` — 스와이프 카운터 추가
4. `SWIPE_MATCH` — SWIPE에서 텍스트만 제거
5. `MIRROR_SWIPE` — REVERSE_SWIPE 변형
6. `MATH_TAP` — COLOR_TAP 구조 + 숫자 버튼
7. `ODD_ONE_OUT` — COLOR_TAP 구조 변형

### Phase 2 - 인지/기억 미션
새 컴포넌트 필요하나 입력은 기존 것 사용.

8. `QUICK_TAP` — 타이머 내 타겟 소멸 로직
9. `CATCH` — 이동 타겟 + 위치 탭 판정
10. `COUNT_TAP` — MULTI_TAP 변형 + 시각 카운팅
11. `PATTERN_TAP` — 순서 검증 로직
12. `SIMON` — 깜빡임 시퀀스 + 탭 순서 검증
13. `FAKE_OUT` — 이중 지시 + 판별

### Phase 3 - 새 입력 방식
InputDetector 확장 필요.

14. `DRAG_TO` — 드래그 시작/끝 좌표 감지
15. `PINCH` — 멀티터치 거리 변화 감지
16. `ROTATE` — 멀티터치 각도 변화 감지
17. `HOLD_AND_TAP` — 동시 터치 상태 추적
18. `DUAL_SWIPE` — 멀티터치 스와이프 감지
19. `RHYTHM` — 타이밍 정밀 판정
20. `COLOR_SWIPE` — 규칙 매핑 UI

## 미션 밸런스 원칙

1. **3초 안에 완료 가능** — 모든 미션은 인지 0.5초 + 행동 2초 이내
2. **즉각 피드백** — 틀리면 바로 실패, 맞으면 바로 성공
3. **텍스트 최소화** — HARD라도 텍스트 10자 이내
4. **첫 경험 직관성** — EASY는 설명 없이 이해 가능해야 함
5. **다중 미션 호환** — 단독 전용 미션은 최소화
